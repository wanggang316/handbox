HandBox 产品需求文档 (PRD)
------

# 1. 产品概述

## 1.1 产品定位
HandBox 是一款面向个人用户的原生化AI工具平台，旨在提供一站式的AI工作环境，让用户能够高效地与多种大语言模型交互，并构建个性化的AI助手。

## 1.2 目标用户
- **AI爱好者**: 希望体验不同AI模型的技术爱好者
- **内容创作者**: 需要AI辅助写作、创意的创作者
- **开发者**: 需要AI辅助编程、调试的程序员
- **研究人员**: 需要AI辅助分析、整理资料的学者
- **知识工作者**: 需要AI提升工作效率的专业人士

# 2. 产品需求

## 2.1 与大模型沟通

![聊天主界面](./resources/chat-normal.png)

### 2.1.1 聊天模型选择

- 支持模型选择，点击聊天输入框内的模型选择，弹出模型选择弹窗，进行模型选择
- 模型及供应商可通过应用设置进行配置

![模型选择](./resources/chat-select-model.png)

### 2.1.2 发送文字及图片

点击聊天输入框内的“+”按钮，选择本地文件，进行发送

### 2.1.3 聊天配置

1. 支持完善的聊天配置，可点击聊天框的设置按钮或聊天框右上角的按钮进行设置
    
    聊天设置包含
    - **系统提示词**（System Prompt）：为聊天定义系统提示词

    ![聊天设置-系统提示词](./resources/chat-settings-prompt.png)

    - **模型参数**：LLM 基础参数配置，包含
        - Temperature
        - Top-P
        - 是否流式输出
        - 最大 Token 数
        - 上下文数

    ![聊天设置-模型参数](./resources/chat-settings-model-args.png)

    - **MCP 设置**：从全局 MCP 读取开启的 MCP 服务器，用户在聊天设置中通过 checkbox 选择，开启的关联到当前聊天

    ![聊天设置-MCP](./resources/chat-settings-mcp.png)

    - **保存 Artifact**：对与可以复用的 聊天配置，可以通过点击“保存为Artifact”将配置进行保存，保存的Artifact会在 Artifact 页面显示

    ![聊天设置-MCP](./resources/chat-settings-new-artifact.png)

### 2.1.4 聊天内容及配置持久化
通过高度结构化、可扩展设计将聊天内容及配置进行本地化存储，方便基于此做上下文工程设计

### 2.1.5 历史消息搜索
通过点击昨天 Sidebar 的搜索按钮，弹出搜索页面，通过输入的关键词在本地索引出关联的聊天会话和消息内容

![聊天-搜索](./resources/chat-search.png)


## 2.2 具备扩展性的聊天界面

![聊天主界面](./resources/chat-normal.png)

1. 支持完整的 Markdown 渲染，含代码高亮、数学公式、mermaid
2. 聊天支持自定义卡片样式，卡片根据内容组件化、可扩展，如推理模型需要的思考过程卡片、MCP 执行过程卡片等。
3. 对于用户消息支持 “复制”、“编辑”操作
4. 对于 AI 消息支持 “复制”、“编辑”、“收藏”、“刷新”、“元数据信息”（耗时、速度、消耗 Token数）
5. 对于推理模型，支持“推理过程”展示
6. 支持调用 MCP 过程展示

## 2.3 设置管理

### 2.3.1 账户

1. 支持未登录正常使用软件，未登录显示缺省头像和登录按钮

![设置-账户-未登录](./resources/settings-account-offline.png)

2. 支持 google 登录，登录后显示用户头像、用户名、邮箱、用户会员状态；点击编辑资料按钮，可以对头像及用户名进行编辑

![设置-账户-已登录](./resources/settings-account-profile.png)
![设置-账户-用户资料编辑](./resources/settings-account-edit.png)

3. 登录/用户管理通过服务端提供接口进行，假定服务端有完善的登录、个人资料获取、退出接口

### 2.3.2 通用

1. 外观样式：跟随系统（默认）、浅色主题、深色主题
2. 语言：简体中文、English
3. 主题色：跟随系统、蓝色、绿色、红色、黄色、紫色、撑死、粉色、棕色
4. 聊天界面自动下滑：默认开启

![设置-通用](./resources/settings-general.png)

### 2.3.3 模型（管理模型供应商和可接入的模型）

1. 供应商管理
    - 供应商分类
        1. 特定主流供应商，包含：OpenAI、Anthropic、Google AI、DeepSeek、OpenRouter
        2. 自定义供应商，包含：兼容 OpenAI API 的供应商、兼容 Anthropic API 的供应商
    - 供应商配置字段
        1. 对于类型 1 特定主流供应商，配置字段为 API Key、Base URL
        2. 对于类型 2 自定义供应商，配置字段为 API Key、Base URL、名称、供应商类型（兼容 OpenAI API、兼容 Anthropic API）
    - 供应商操作
        1. 对于类型 1 特定主流供应商，支持检测（通过简单的探活消息 确认key 是否生效）、供应商开关操作（打开后聊天页面可选择该供应商的模型）
        2. 对于类型 2 自定义供应商，支持检测（通过简单的探活消息 确认key 是否生效）、供应商开关（打开后聊天页面可选择该供应商的模型）、删除操作
2. 模型管理
    - 通过供应商管理页面的 “获取模型列表” 按钮获取该供应商的所有模型，用户选择的模型可以在聊天页面可进行选择使用
    - 模型选择页面支持简单的模型搜索功能，模糊搜索模型名称和id
3. 页面交互方式
    - 供应商列表页，此为模型设置的入口页，展示供应商条目及配置状态，底部为添加供应商按钮

      ![设置-模型-供应商列表](./resources/settings-model-provider-list.png)

    - 点击供应商列表页面底部的“添加供应商”按钮，弹出添加供应商页面，填写 供应商名称、类型、图标，点击“确认”进入供应商配置页面进行配置

      ![设置-模型-新建自定义供应商](./resources/settings-mode-new-custom-provider.png)

    - 供应商配置页，可以配置供应商的字段、操作供应商可执行的动作

      ![设置-模型-供应商设置](./resources/settings-model-provider-settings.png)

    - 点击供应商配置页的 “获取模型列表” 按钮，弹出模型选择页，用户选择完成后，点击确认或取消关闭页面

      ![设置-模型-模型选择](./resources/settings-model-select-model.png)

## 2.4 Artifact 管理

Artifact 指一套完整的聊天配置（系统提示词、MCP、模型参数）的合集，方便后续进行同样配置的聊天

页面展示及交互方式：
- 点击左侧 Sidebar 菜单进入页面，Artifact通过卡片展示
- 点击列表中的某一个 Artifact，弹窗展示聊天配置页面
- 点击卡片上的“使用” 按钮，则基于此 Artifact 新建聊天

![Atrifact](./resources/artifacts.png)

# 3. 非功能性需求

## 3.1 性能要求
- 应用启动时间 < 3秒
- 对话响应时间 < 2秒（不包括LLM响应时间）
- 支持同时处理10个以上的对话会话
- 内存使用 < 500MB（空闲状态）

## 3.2 安全要求
- API密钥本地加密存储
- 代码执行环境沙箱隔离
- 用户数据不上传到云端
- 支持数据备份和恢复

## 3.3 可用性要求
- 直观的用户界面，学习成本低
- 支持键盘快捷键操作
- 提供操作指南和帮助文档
- 错误信息清晰易懂

## 3.4 兼容性要求
- 支持Windows、macOS、Linux三大平台
- 支持不同屏幕分辨率和DPI
- 向后兼容数据格式

这个PRD将作为开发团队的指导文档，确保产品开发方向明确，功能完整，用户体验良好。
