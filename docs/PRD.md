# HandBox 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
HandBox 是一款面向个人用户的原生化AI工具平台，旨在提供一站式的AI工作环境，让用户能够高效地与多种大语言模型交互，并构建个性化的AI助手。

### 1.2 目标用户
- **AI爱好者**: 希望体验不同AI模型的技术爱好者
- **内容创作者**: 需要AI辅助写作、创意的创作者
- **开发者**: 需要AI辅助编程、调试的程序员
- **研究人员**: 需要AI辅助分析、整理资料的学者
- **知识工作者**: 需要AI提升工作效率的专业人士

### 1.3 核心价值主张
- **多模型集成**: 一个平台管理多个AI模型
- **本地化存储**: 数据安全，隐私保护
- **个性化定制**: 可自定义Prompt和Agent
- **专业工具**: 面向专业用户的高级功能

## 2. 功能需求
1. 与大模型沟通
    1. 主流 LLM API 接入（OpenAI/Anthropic/Google AI/DeepSeek/OpenRouter）
        1. 用户可以随时选择使用的模型
        2. 
    2. 聊天配置支持以下
        1. 系统提示词（System Prompt）：为聊天定义系统提示词
        2. 模型参数：LLM 基础参数配置，包含 
            - Temperature
            - Top-P
            - 是否流式输出
            - 最大 Token 数
            - 上下文数
        3. MCP 设置：从全局 MCP 读取开启的 MCP 服务器，用户在聊天设置中选择，开启的关联到当前聊天
    3. 聊天内容及配置持久化，将上下文消息传给大模型
    4. 支持历史消息搜索，可以关键词在本地索引出关联的会话和消息内容

1. 具备扩展性的聊天界面
    1. 支持完整的 Markdown 渲染，含代码高亮、数学公式、mermaid
    2. 聊天支持自定义卡片样式，卡片根据内容组件化、可扩展，如推理模型需要的思考过程卡片、MCP 执行过程卡片等。
    3. 对于用户消息支持 “复制”、“编辑”操作
    4. 对于 AI 消息支持 “复制”、“编辑”、“收藏”、“刷新”、“元数据信息”（耗时、速度、消耗 Token数）
    5. 对于推理模型，支持“推理过程”展示
    6. 支持调用 MCP 过程展示
2. 设置管理
    1. 账户
        1. 支持未登录正常使用软件，未登录显示缺省头像和登录按钮
        2. 支持 google 登录，登录后显示用户头像、用户名、邮箱、用户会员状态
        3. 登录/用户管理通过服务端提供接口进行，假定服务端有完善的登录、个人资料获取、退出接口
    2. 通用
        1. 外观样式：跟随系统（默认）、浅色主题、深色主题
        2. 语言：简体中文、English
        3. 主题色：跟随系统、蓝色、绿色、红色、黄色、紫色、撑死、粉色、棕色
        4. 聊天界面自动下滑：默认开启
    3. 模型（管理模型供应商和可接入的模型）
        1. 供应商管理
            1. 供应商分类
                1. 特定主流供应商，包含：OpenAI、Anthropic、Google AI、DeepSeek、OpenRouter
                2. 自定义供应商，包含：兼容 OpenAI API 的供应商、兼容 Anthropic API 的供应商
            2. 供应商配置字段
                1. 对于类型 1 特定主流供应商，配置字段为 API Key、Base URL
                2. 对于类型 2 自定义供应商，配置字段为 API Key、Base URL、名称、供应商类型（兼容 OpenAI API、兼容 Anthropic API）
            3. 供应商操作
                1. 对于类型 1 特定主流供应商，支持检测（通过简单的探活消息 确认key 是否生效）、供应商开关操作（打开后聊天页面可选择该供应商的模型）
                2. 对于类型 2 自定义供应商，支持检测（通过简单的探活消息 确认key 是否生效）、供应商开关（打开后聊天页面可选择该供应商的模型）、删除操作
        2. 模型管理
            1. 通过供应商管理页面的 “获取模型列表” 按钮获取该供应商的所有模型（弹出页面）
            2. 在弹出的模型列表页面进行选择（该页面支持简单的模型搜索功能，模糊搜索模型名称和 id），通过checkbox选择聊天可以使用的模型，未选择的不再聊天页面漏出
        3. 页面交互方式
            1. 第一级页面显示供应商列表，供应商条目显示供应商配置状态，底部为添加供应商按钮。点击供应商列表的条目跳转到二级页面，点击添加供应商按钮录入 供应商名称、类型、图标后进入供应商详情页面进行配置
            2. 第二级显页面示供应商详情，可以配置供应商的字段、操作供应商可执行的动作
    4. MCP
        1. 展示 MCP 服务器列表，每个条目有一个开关，用户操作是否打开
        2. 列表底部有 “管理MCP 服务器”按钮，点击后弹出 “编辑 MCP 服务器” JSON 文件编辑器
    5. 快捷键
        1. 发送消息：Enter、Command+Enter
    6. 关于
        1. 检查更新：显示当前版本，点击本行后检查更新
        2. 更新日志：通过浏览器跳转到官网 changelog 页面
        3. 官方网站：通过浏览器跳转到官网主页
        4. 邮箱、Twitter 链接，展示页面底部，点击后复制邮箱地址和跳转到 Twitter 主页

## 3. 非功能性需求

### 3.1 性能要求
- 应用启动时间 < 3秒
- 对话响应时间 < 2秒（不包括LLM响应时间）
- 支持同时处理10个以上的对话会话
- 内存使用 < 500MB（空闲状态）

### 3.2 安全要求
- API密钥本地加密存储
- 代码执行环境沙箱隔离
- 用户数据不上传到云端
- 支持数据备份和恢复

### 3.3 可用性要求
- 直观的用户界面，学习成本低
- 支持键盘快捷键操作
- 提供操作指南和帮助文档
- 错误信息清晰易懂

### 3.4 兼容性要求
- 支持Windows、macOS、Linux三大平台
- 支持不同屏幕分辨率和DPI
- 向后兼容数据格式

## 4. 用户场景

### 4.1 场景一：内容创作工作流
**用户角色**: 技术博客作者
**场景描述**: 
1. 使用"技术写作助手"Agent帮助构思文章主题
2. 利用Prompt模板生成文章大纲
3. 通过对话系统完善文章内容
4. 使用代码执行功能验证技术示例

### 4.2 场景二：数据分析任务
**用户角色**: 数据分析师
**场景描述**:
1. 创建专门的数据分析Agent
2. 上传数据文件，Agent自动分析数据特征
3. 通过自然语言描述分析需求
4. Agent生成并执行分析代码，产生可视化图表

### 4.3 场景三：学习和研究
**用户角色**: 研究生学生
**场景描述**:
1. 创建学术研究Agent，配置专业知识库
2. 批量导入论文和资料到长期记忆
3. 通过对话进行文献综述和观点整理
4. 利用思维链功能进行复杂问题分析

## 5. 产品优先级

### 5.1 MVP功能（第一版本）
1. 基础LLM API集成（OpenAI、Claude）
2. 简单对话系统
3. 基础Prompt管理
4. 数据本地存储

### 5.2 第二版本
1. 完整的Agent创建和管理
2. 记忆系统基础功能
3. 基础工具集成
4. UI/UX优化

### 5.3 后续版本
1. 高级记忆和检索功能
2. MCP协议集成
3. 代码执行环境
4. 插件系统
5. 数据可视化和分析工具

## 6. 成功指标

### 6.1 用户指标
- 日活跃用户数
- 用户留存率（7天、30天）
- 用户会话时长

### 6.2 功能指标
- Prompt模板使用频率
- Agent创建和使用率
- API调用成功率
- 功能使用分布

### 6.3 技术指标
- 应用稳定性（崩溃率）
- 响应时间
- 内存和CPU使用率

## 7. 风险和缓解策略

### 7.1 技术风险
- **API限制和费用**: 实施用量监控和预警机制
- **数据安全**: 采用本地加密存储，定期安全审计
- **跨平台兼容**: 充分测试各平台功能

### 7.2 产品风险
- **用户学习成本**: 提供详细文档和教程
- **功能复杂性**: 采用渐进式功能展示
- **竞品压力**: 专注于差异化特性和用户体验

这个PRD将作为开发团队的指导文档，确保产品开发方向明确，功能完整，用户体验良好。
